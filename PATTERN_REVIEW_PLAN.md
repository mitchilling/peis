# Weekly Pattern Review

The real collapse of this system begins when:
> multiple patterns silently start governing the same design space

In order to avoid that, we shall start a regular overlap review ritual, with the following 4 questions:

## 1. Duplicate

> Are two patterns describing the same stable law?

If yes:

> merge or parent/sub-pattern

## 2. Boundary

> Where does each pattern stop being useful?

This prevents “pattern imperialism”.

## 3. Precedence

> If two patterns trigger together, which one wins?

This is huge.

Example:

- generic pipeline says “use typed APIs”
- preference/effective says “separate semantics”

Which is primary?

Sometimes:

> semantic separation outranks access style

That precedence matters later for agent reasoning.

## 4. Contradiction

> Did a recent case disprove or weaken an older pattern?

If yes:

- refine
- narrow scope
- add trade-off section
- downgrade certainty
